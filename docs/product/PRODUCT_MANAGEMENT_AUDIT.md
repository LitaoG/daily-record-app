# 产品经理交付完整性审计

状态：M0 gaps identified and prioritized
更新时间：2026-07-16

本文回答两个问题：现阶段产品经理应该完成什么，以及哪些证据还不足以支持大规模 Android UI 开发。审计以[开发准备度基线](IMPLEMENTATION_READINESS.md)、仓库五张视觉原型、GitHub Issues 和当前文档为事实来源。

## 1. 结论

Daily Record 已经具备可以启动数据模型与本地持久化工作的产品基础，但还不能把视觉原型当作无歧义的 UI 实现规范。

当前最重要的工作不是继续增加功能，而是完成三件事：

1. 统一 P0 范围和所有文档，消除登录、同步、热力图、深色模式等范围漂移。
2. 修复原型中的颜色语义、空日期、时长输入、撤销和错误反馈等实现歧义。
3. 取得真实目标用户证据，完成问题访谈和 5 人可用性测试，再决定是否扩大功能面。

## 3. 事实源优先级

发生冲突时按以下顺序处理，并同步修复低优先级来源：

1. 已接受 ADR 与[开发准备度基线](IMPLEMENTATION_READINESS.md)。
2. [产品策略](PRODUCT_STRATEGY_V1.md)、[用户故事](USER_STORIES_AND_ACCEPTANCE.md)和[统计口径](../STATISTICS.md)。
3. [界面与交互](../UI_UX.md)、[信息架构](INFORMATION_ARCHITECTURE.md)和可扩展标记规范。
4. 视觉原型、GitHub Issue 和宣传型一页图。

视觉原型不是数据规则的来源；如果原型与已冻结规则冲突，先修原型，不修改业务契约迎合图片。

## 4. 当前阻断性问题

### P0-1 活动颜色与状态颜色冲突

活动色必须表示活动身份；`DONE/MISSED/SKIPPED/UNSET` 通过图标、边框、纹理和文字表达。当前标记系统图片把橙/绿/灰解释为状态，且手冲跨页面使用不同颜色，不能直接交给开发实现。

### P0-2 空日期语义不明确

月历中反复出现 `0` 胶囊会制造噪声，也容易把“没有任何明确记录”理解成“明确完成 0 项”。P0 默认空日期不显示汇总胶囊；只有选中日期的详情或无障碍描述说明“未记录”。组件压力测试可以保留 0 场景，但不能把它当成默认月历表现。

### P0-3 输入、撤销与失败反馈不完整

- COUNT：`+1/-1`，点击数值进入精确输入。
- DURATION：默认 `+5/-5` 分钟，点击数值精确输入非负整数分钟。
- BOOLEAN：做了、没做、跳过和清除。
- 本地保存成功后给轻量反馈；误操作可以立即撤销。
- 本地保存失败必须保留编辑值、说明未保存并提供重试，不能显示成功状态。

### P0-4 研究证据不足

当前证据主要来自创意提出者和开源项目，尚不能证明目标用户愿意长期使用“日历 + 混合计量”结构。数据模型可以开始，但扩大 UI 和高级功能前必须执行[发现研究计划](DISCOVERY_RESEARCH_PLAN.md)与[可用性测试计划](USABILITY_TEST_PLAN.md)。

### P0-5 统计固定数据集不一致

统计 PNG 的月次数合计为 118、月天数合计为 78，与顶部 128 次、74 天冲突；演示参考日期也未明确。该图片只能用于结构评审，不能作为统计实现或测试真值。修订时必须让汇总卡、月表和趋势图共享同一固定数据集，并校验记录日均的舍入。

## 5. 阶段门槛

### M0 Product Ready

- P0 文档无冲突。
- 阻断性原型语义已修复或明确记录为 Figma/Compose 前置债务。
- 至少 5 名目标用户完成可用性测试并达到既定门槛。
- 高风险均有缓解动作。

### M1 Local Alpha

- #5 → #6 → #4 → #7 按顺序交付。
- 飞行模式、进程重建、迁移和固定统计数据集通过。
- 核心 TalkBack 流程可以完成。

### M2 Private Beta

- 导入导出和恢复演练通过。
- 生产指标口径、崩溃监控、隐私说明和反馈渠道就绪。
- Beta 数据用于校准 WVU、激活和留存假设。

## 6. 当前不阻塞 P0 的未决项

- 正式展示名称与商店关键词。
- 中国大陆或其他地区的云端部署区域。
- 邮箱、第三方或其他登录方式。
- 云同步商业成本、端到端加密和账户合并。
- iOS/Web、多语言、提醒、目标、连续天数和 Health Connect。

这些问题必须在对应阶段前形成 ADR，但不能反向阻塞本地数据模型和离线记录闭环。
