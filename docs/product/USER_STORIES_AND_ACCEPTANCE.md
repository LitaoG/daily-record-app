# 用户故事与验收标准

状态：P0 planning baseline
更新时间：2026-07-16

## Epic A：首次使用

### US-A1 直接开始记录

作为首次用户，我希望不注册账号也能进入日历，以便立即判断应用是否适合我。

验收：

- 首次启动说明本地存储与隐私原则。
- 用户可选择默认的手冲、健身模板，或跳过模板。
- 不登录也能完成 P0 的所有本地功能。
- 首次流程结束后进入当前月份，今天清晰可见。

## Epic B：日历与日期

### US-B1 查看一个月

作为用户，我希望从一个稳定的月历看到每天的活动，以便快速回顾。

验收：

- 月份标题、前后切换和回到今天始终可用。
- 今天、选中日期和活动标记互不覆盖。
- 同日始终只有一个汇总胶囊，最多 3 色段并显示活动总数；8 项及以上显示 `8+`。
- `UNSET` 与 `MISSED` 不能使用同一种表现。
- 一周起始日支持星期一或星期日，默认星期一。

### US-B2 修改历史记录

作为用户，我希望点击任意过去日期并使用与今天相同的编辑规则，以便纠正遗漏。

验收：

- 日期详情清楚显示正在编辑的本地日期。
- 修改后月历和统计立即更新，不等待网络。
- 跨月、跨年和闰日均可编辑。

## Epic C：活动记录

### US-C1 记录手冲次数

作为手冲用户，我希望一次点击增加 1 次，也能直接改成正确次数，以便快速且准确地记录。

验收：

- 默认主操作为 `+1`，并提供 `-1` 与直接输入。
- quantity 只允许非负整数。
- quantity 从正数改为 0 后，状态变为 `MISSED`。
- 清除后状态变为 `UNSET`，quantity 为空。
- 同一活动同一天的总次数只有一个权威值。

### US-C2 记录健身状态

作为健身用户，我希望明确标记做了、没做或跳过，以便回顾时不把“忘记记录”误判为“没做”。

验收：

- 提供 `DONE`、`MISSED`、`SKIPPED` 和清除。
- `DONE` 计入活动天数，其他状态不计入。
- 跳过不会显示为失败，也不自动参与未来完成率。

### US-C3 管理活动

作为用户，我希望编辑活动名称、颜色、排序和归档，以便后续加入新的生活记录。

验收：

- 活动包含名称、颜色、图标、计量方式和单位。
- 有历史记录时改变计量方式必须禁止或经过显式迁移。
- 归档后不出现在默认记录区，但历史和统计保留。
- P0 不提供轻易永久删除活动的入口。

## Epic D：统计

### US-D1 查看可信汇总

作为用户，我希望查看手冲总次数、手冲天数和健身天数，以便了解真实积累。

验收：

- 总次数等于未删除每日 quantity 之和。
- 活动天数等于 `quantity > 0` 或 `status == DONE` 的不同本地日期数。
- 选择单一活动后，汇总卡、表格和图表不得混入其他活动的数据。
- 只有“全部活动”视图允许显示多个活动的摘要，且每个指标必须明确活动名称。
- 周、月、年均可切换到过去周期。
- 当前未结束周期标注“进行中”。
- 空数据展示 0 和解释，不使用 `--` 混淆未知。

### US-D2 查看精确表

作为用户，我希望周、月、年都有精确表格，以便从图表回到实际数字。

验收：

- 周表展示连续 7 天。
- 月表展示自然月每日记录与合计。
- 年表展示 12 个月的次数和活动天数。
- 所有周期使用 `[startDate, endExclusive)`。
- 表格与日历点击可回到对应日期。

## Epic E：可靠性与隐私

### US-E1 离线记录

作为用户，我希望无网时仍能正常记录，以便任何场景都不丢数据。

验收：

- 飞行模式下新增、修改和清除均成功。
- 杀死进程并重启后记录仍存在。
- 同步失败不回滚本地已显示记录。

### US-E2 保护私人数据

作为用户，我希望默认不把行为内容发送到第三方，以便放心长期使用。

验收：

- 日志不输出活动名称、备注、记录数值或账号令牌。
- 分析事件默认不包含上述内容。
- 导出、删除和未来同步都明确说明数据范围。

## P0 端到端验收场景

1. 飞行模式启动应用，选择手冲和健身模板。
2. 今天手冲 `+1` 两次，健身标记 `DONE`。
3. 昨天手冲直接输入 1，健身标记 `MISSED`。
4. 后天健身标记 `SKIPPED`，确认不计入活动天数。
5. 验证总手冲次数为 3、手冲天数为 2、健身天数为 1。
6. 在周、月、年表中验证同一结果。
7. 重启应用并再次验证；归档健身后历史统计仍存在。
