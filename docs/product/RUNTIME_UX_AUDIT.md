# 运行时 UX 与统计验收

日期：2026-07-19

设备：Pixel 4 API 34 模拟器，1080×2280，正常字体

范围：仅手冲月历、记录结果、周/月/年/全部统计

## 总体结论

当前版本可以进入真实设备人工验收。核心路径没有发现阻断项：历史日期有快速入口，0 次与未填写明确区分，月周明细会随月份重建，所有统计口径能从逐日记录反算。

## 审计步骤

1. **月历与快速历史入口——健康。** [月历截图](assets/hand-brew-v2/13-android-calendar-audit-api34.png)显示 0、1、2 次使用文字、明暗和边框共同表达；大号年月有“点此快速跳转”，相邻月份箭头和“今天”保留快速操作。
2. **周统计汇总——健康。** [周汇总截图](assets/hand-brew-v2/18-android-statistics-seeded-api34.png)由 0、1、2 次记录得到 3 次、2 天、记录日均 1.5；没有把 0 次计入手冲天数。
3. **周统计明细——健康。** [逐日明细截图](assets/hand-brew-v2/19-android-statistics-details-api34.png)同时展示未填写、0 次、1 次和 2 次，用户可以直接反查汇总来源。
4. **月份切换——健康。** [6 月周明细截图](assets/hand-brew-v2/15-android-statistics-month-switch-api34.png)的五行日期范围完全按 6 月重建；切回 7 月后第 3 周恢复 3 次、2 天，没有遗留 6 月标签或数字。
5. **年度统计——健康。** [年度截图](assets/hand-brew-v2/16-android-statistics-year-api34.png)明确显示“截至 7 月19日”，未来月份保持空值，不制造预测。
6. **全部历史空态——健康。** [全部历史截图](assets/hand-brew-v2/17-android-statistics-all-api34.png)解释为什么暂无统计，并提供唯一明确的“去日历记录”入口。

## 优点

- 日历和统计共享同一浏览日期，用户不会在切页后被带回今天。
- 所有总数旁都有天数或单位，统计卡、明细行和月历汇总使用同一口径。
- 颜色不是唯一状态信号；日期格和统计行都有文字，UI 语义树包含完整日期、次数、天数和按钮用途。
- 空数据不显示伪趋势或预测，减少误解。

## 剩余风险与边界

- 当前截图可以证明布局、层级和可见状态，不能代替真机 TalkBack 朗读顺序、触觉反馈和不同厂商字体渲染测试。
- 云恢复的自动化与生产冒烟已经通过，但双实体设备、目标移动网络和长时间后台恢复仍需人工验收。
- Pixel 7 AVD 的 Android System UI 冷启动故障不是应用错误；仍建议真机覆盖至少一台窄屏和一台大屏设备。

## 暂缓项

- 滑动换月、全年热力图和趋势图只有在真机反馈证明当前入口不足时再做。
- 连续天数、目标、提醒、健身和其他活动不属于仅手冲范围。
