# KPI 与测量框架

状态：Provisional until beta baseline
更新时间：2026-07-16

## 1. 测量原则

- 指标用于验证用户是否持续获得记录与回顾价值，不用于制造打卡压力。
- 产品尚无生产基线，所有目标值都是内测假设，必须在 5 人可用性测试和首轮 Beta 后校准。
- 默认仅做本地计算；如果未来加入远端分析，必须获得明确同意并进行数据最小化。
- 活动名称、备注、具体记录数值和用户导出内容不进入遥测。

## 2. 北极星候选

| 指标 | 定义 | 为什么重要 | 初始观察周期 |
|---|---|---|---|
| 周价值用户 WVU | 滚动 7 天内，在至少 3 个不同本地日期产生一条 `DONE` 或 quantity > 0 的用户 | 同时代表记录低摩擦与持续价值 | 每周 |
| 4 周留存记录者 | 首次价值周后的第 4 周仍至少产生 1 个有效记录日的用户占比 | 验证产品是否成为长期工具 | 每 4 周 |

首个 Beta 阶段以 WVU 为主要运营指标，4 周留存作为滞后结果指标。两者不在纯本地 MVP 阶段强制上报。

## 3. 激活指标

- 首次会话完成第一笔有效记录的比例。
- 首次会话创建或保留至少一个活动的比例。
- 首次 24 小时内回看日期或打开统计的比例。
- 首次使用中因为登录要求而退出的比例，目标为 0，因为 P0 不要求登录。

## 4. 体验驱动指标

| 指标 | 内测目标 | 测量方法 |
|---|---:|---|
| 打开到完成常用记录的中位时长 | ≤ 5 秒 | 现场任务录像或本地时间点 |
| 常用记录点击数 | ≤ 3 次 | 原型路径审查 |
| 核心任务完成率 | ≥ 90% | 至少 5 名目标用户 |
| 多活动日期识别正确率 | ≥ 80% | 可用性测试提问 |
| 统计口径理解率 | ≥ 80% | 让用户解释总次数与总天数 |

## 5. 质量护栏

- 严重数据丢失或无法解释的数据一致性缺陷：0。
- 固定统计数据集正确率：100%。
- Crash-free sessions：Beta 目标 ≥ 99.5%。
- 本地记录交互响应：数据库初始化后目标 < 100ms。
- 无障碍：颜色不是唯一编码；核心控件满足 Material 触控尺寸；TalkBack 可读。
- 隐私：公开日志、仓库和默认分析事件中的真实活动内容为 0。

## 6. 事件字典草案

以下事件只记录行为类别和性能，不记录活动内容：

| 事件 | 允许字段 | 禁止字段 |
|---|---|---|
| `app_opened` | session、冷/热启动、耗时 | 账号、活动内容 |
| `date_selected` | 日期相对类型：today/past/future | 具体日期 |
| `record_created` | measurementType、tapCount、durationMs | 活动名、quantity、备注 |
| `record_updated` | measurementType、durationMs | 旧值、新值 |
| `record_cleared` | measurementType | 被清除内容 |
| `stats_viewed` | periodType、entryPoint | 精确统计值 |
| `activity_created` | measurementType | 名称、颜色的用户语义 |
| `export_completed` | format、success | 文件内容与路径 |
| `sync_conflict` | conflictType、resolved | 记录内容 |

## 7. 决策阈值

- 若核心记录中位时长 > 5 秒，先优化日期详情与默认动作，不增加新功能。
- 若多活动日期识别正确率 < 80%，重新设计色点、图例和筛选。
- 若统计理解率 < 80%，先改口径说明与表格层级，不增加更多图表。
- 若出现任何不可恢复的数据丢失，暂停功能迭代，优先修复持久化与迁移。
