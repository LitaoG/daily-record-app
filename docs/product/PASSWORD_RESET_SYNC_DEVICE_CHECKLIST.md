# 找回密码与同步真机验收清单

目标：PR 合并后只验证真实设备、真实邮箱和生产网络差异，不重复模拟器已经覆盖的内部实现测试。

## 准备

- 从远端 `main` 构建并安装 Debug 包；确认 `app/google-services.json` 只存在本机且没有加入 Git。
- 准备一个专用测试邮箱和两台 Android 手机；不要使用重要主账号。
- 第一台手机先在未登录模式记录两个历史日期：一个正次数、一个明确 0 次。

## 单机密码重置

1. 登录页输入测试邮箱，点击“忘记密码？”，确认弹窗预填同一邮箱。
2. 输入错误邮箱格式，确认发送禁用并显示“请输入有效邮箱”。
3. 连续快速点击发送，确认只进入一次发送中状态，没有重复提示或多个页面。
4. 检查收件箱和垃圾邮件，打开 Firebase 重置链接并设置新密码。
5. 返回应用，用旧密码登录应失败，用新密码登录应成功；原本机两条记录应立即出现。
6. 对一个未注册邮箱发送重置，应用必须显示与已注册邮箱完全相同的成功文案。

## 中国大陆网络与恢复

1. 关闭代理/VPN，确认日历仍能新增、修改和清除记录，应用不崩溃且显示待同步或离线。
2. 在仍处于登录状态时开启代理/VPN，不重启应用；等待同步状态自动变为“已同步”，必要时仅使用一次“立即同步”。
3. 关闭并重开应用，确认刚才的记录仍由 Room 立即显示，不等待云端。

## 双机与账号隔离

1. 第二台手机登录同一测试账号，确认正次数、明确 0 次和清除墓碑都能恢复。
2. 第一台离线修改一个日期，第二台在线修改另一个日期；第一台恢复网络后，两台最终显示相同结果。
3. 退出测试账号并登录另一个账号，确认看不到前一个账号的任何日历或统计记录。
4. 再登录原测试账号，确认数据恢复且没有被另一个账号覆盖。

## 可访问性与系统边界

- 把系统字体设为最大档，确认登录页可滚动，找回密码弹窗标题、邮箱、发送和取消均可见。
- 开启 TalkBack，依次检查“打开重置密码”、邮箱错误、发送中、成功提示和返回登录的朗读顺序。
- 记录手机品牌、Android 版本、邮箱客户端、浏览器、网络环境和任何异常截图。

验收完成后把缺陷按“设备/系统/网络/步骤/预期/实际”发回；本阶段不创建正式 Release。
