# 可扩展日历标记系统

状态：Product rule v1
更新时间：2026-07-16

## 目标

让月历在 0、2、4、8+ 项活动下都保持可读，并保证新增活动只需要创建活动配置，不需要重做日期单元格。

![可扩展标记系统](assets/prototype-v1/05-scalable-marker-system.png)

## 显示规则

### 全部活动概览

- 每个日期只渲染一个 `ActivitySummaryCapsule`。
- 色段只取当天可见活动中按用户排序最靠前的 3 项。
- 右侧数字显示当天有记录或明确状态的活动总数；8 项及以上显示 `8+`。
- `0` 表示当天没有有效记录或明确状态，不等于 `MISSED`。
- 色段只帮助快速识别，不承诺在概览中表达全部活动。

### 单活动筛选

- 胶囊显示活动图标、精简名称（空间允许时）、状态符号和数量。
- 手冲图标固定为飞机。
- 次数型显示整数；时长型显示精简时长；布尔型显示状态。
- 状态编码：
  - `UNSET`：空心轮廓。
  - `DONE`：勾 + 实色填充。
  - `MISSED`：叉 + 斜线纹理。
  - `SKIPPED`：短横线 + 低对比填充。

### 日期详情

- 展示当天全部活动，不做 3 项截断。
- 支持次数、完成/未完成/跳过、时长以及后续扩展的测量类型。
- 清空记录返回 `UNSET`；明确记录 0 次或“没做”进入 `MISSED`。
- 归档活动仍可在历史日期中读取，但默认不出现在今天的新增列表中。

## 排序与聚合

1. 用户自定义排序优先。
2. 同一排序下，已有明确记录的活动优先于 `UNSET`。
3. 汇总数字统计“有记录或明确状态”的活动，不统计仅存在于活动库但当天未设置的活动。
4. 色段不得被解释为数量占比；它们只是最多 3 个活动的身份提示。

## 可访问性

- 任何状态都不能只靠颜色表达。
- TalkBack 日期描述示例：`7月16日，4项活动，手冲2次，健身已完成，另有2项`。
- 触控目标不少于 Material 建议尺寸；胶囊本身是日期格的一部分，不形成过小的独立按钮。
- 高对比模式下保留轮廓、符号和数字，允许弱化或关闭纸张纹理。

## 压力测试验收

| 场景 | 必须证明的结果 |
|---|---|
| 0 项活动 | 显示空态，不误导为“没做” |
| 2 项活动 | 两项可快速辨识，汇总数为 2 |
| 4 项活动 | 最多 3 色段，汇总数为 4，无第四个点 |
| 8+ 项活动 | 仍只有一个胶囊，显示 `8+`，日期数字不被挤压 |
| 单活动筛选 | 图标、状态、数量可独立理解 |
| 归档活动 | 历史日期和统计仍可解释 |

## 实现边界

- UI 组件只接收聚合后的日期摘要，不直接读取某个固定活动字段。
- 数据模型以 `activityId` 关联记录，不增加 `coffeeDone`、`fitnessDone` 等专属列。
- 新增活动不新增底部导航项；它只出现在活动管理、筛选和日期详情中。
- 概览最多 3 色段是稳定产品规则，不随活动数量增长。
