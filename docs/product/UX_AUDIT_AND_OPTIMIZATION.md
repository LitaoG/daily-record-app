# P0 原型体验审计与优化清单

状态：Blocking findings recorded
更新时间：2026-07-16

审计范围：仓库中的五张 390 × 844 视觉基线及标记系统说明。这里只评价截图中可见的流程、信息层级与可访问性风险；真实点击、键盘、动画、TalkBack 和性能必须在可交互原型或 Android 构建中另测。

## 总体判断

核心结构是成立的：月历是主入口，日期详情兼容多种计量方式，统计页把精确表格放在趋势图之前，活动管理没有为手冲或健身建立专属结构。

当前不能直接照图实现的关键原因是颜色语义不一致。修复顺序固定为：活动身份色 → 状态编码 → 空日期 → 输入与撤销 → 错误/空状态 → 视觉润色。

## 1. 月历概览：需优化

![月历概览](assets/prototype-v1/01-calendar-overview.png)

可保留：

- 当前月、今天、选中日期和活动筛选层级清晰。
- 单个汇总胶囊在 2/4/8+ 活动下比无限堆点稳定。
- 选中日期直接露出常用动作，适合 5 秒记录目标。

必须调整：

- 空日期默认不显示 `0` 胶囊，避免把 `UNSET` 误解为 `MISSED` 或“完成 0 项”。
- 色段只表示活动身份；状态用勾、叉、短横线、虚线边框或纹理补充。
- 筛选后的日期格显示活动图标、状态和数量，不继续使用“活动总数”语义。
- 手冲的基础色在月历、详情、统计、活动管理中必须相同。

可访问性风险：日期格信息密度高；需验证 200% 字体、TalkBack 顺序和 48dp 触控目标，不能从截图推断合规。

## 2. 日期详情：基本健康

![日期详情](assets/prototype-v1/02-date-details.png)

可保留：

- COUNT、BOOLEAN、DURATION 与 `SKIPPED/UNSET` 可以共存。
- 活动行的大图标和文字比纯色点更易扫描。
- 未记录、没做和跳过已经有文字及形状差异。

必须调整：

- 数值本身可点击进入精确输入，并给出编辑提示；否则用户只能猜直接输入入口。
- DURATION 默认以 5 分钟步进，点数值可精确输入；单位与活动配置保持一致。
- `+/-` 和状态切换即时保存后提供短暂撤销；数据库失败时显示“未保存，重试”，不能假装成功。
- 清除记录必须是次要动作并解释会回到“未记录”。
- 6 项活动同屏只是压力场景；真实顺序应尊重用户排序，常用活动置前。

可访问性风险：状态分段按钮需逐项可聚焦并朗读“健身，已完成/未完成/跳过”；不能把整卡和内部按钮都做成重复点击目标。

## 3. 统计页：结构健康，数据存在阻断

![统计页](assets/prototype-v1/03-statistics.png)

可保留：

- 单活动筛选后只显示手冲总次数、手冲天数和记录日均，已修正跨活动污染。
- 精确表格优先于趋势图，符合可信统计原则。

必须调整：

- 当前月次数合计为 118，不等于汇总卡 128；月天数合计为 78，不等于汇总卡 74。汇总卡、月表和趋势图必须改用同一固定数据集。
- 明确演示参考日期；只有参考日期之后的月份为空，趋势线不得为未来月份画实线或预测。
- 时间范围文案使用“全部历史”，避免与“全部活动”混淆。
- 活动选择器和时间选择器在视觉上保持不同控件形态。
- 趋势图必须提供同数据表或可访问摘要；空数据时不渲染误导性的坐标线。
- 当前未结束周期继续显示“进行中”，并解释未来月份为空不是缺失错误。

## 4. 活动管理：基本健康

![活动管理](assets/prototype-v1/04-activity-management.png)

可保留：

- 名称、图标、计量类型和颜色在一行内可扫描。
- 排序和新增入口明显，能支撑 8+ 活动。

必须调整：

- 计量类型统一显示“次数 / 完成 / 时长”，不能在文档、原型和代码中出现不同命名。
- 有历史记录时禁止直接改变计量类型，并解释“归档并新建”的替代路径。
- 活动归档后进入独立折叠区，提供撤销；P0 不暴露永久删除。
- 新建活动必须预览其在日历汇总胶囊和单活动筛选中的表现。

## 5. 标记系统：存在阻断性冲突

![标记系统](assets/prototype-v1/05-scalable-marker-system.png)

必须修正后才能成为实现规范：

- 图片中的“橙色=未完成、绿色=已完成”与冻结契约冲突。色段必须表示活动身份，而不是状态。
- 状态编码固定为：`UNSET` 虚线/空心，`DONE` 勾，`MISSED` 叉和纹理，`SKIPPED` 短横线和中性色；文字或 TalkBack 描述始终存在。
- 全部活动概览只汇总有明确记录的活动；当天没有明确记录时默认不显示胶囊。
- 单活动筛选可以显示该活动基础色，但完成状态仍由符号和文字表达。
- 0 场景只用于组件压力测试和详情空状态，不在月历上批量渲染 `0`。

## 跨屏一致性规则

1. `Activity.color` 是身份色，跨页面不变。
2. `RecordStatus` 不拥有新的活动身份色；它只改变符号、边框、纹理和文案。
3. 手冲图标固定为飞机；统计中的“记录日均”使用中性统计图标，避免被误认成另一个手冲指标。
4. 所有“全部”明确写成“全部活动”或“全部历史”。
5. 所有写操作具有成功、失败和撤销反馈。
