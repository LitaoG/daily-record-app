# 视觉方向与原型决策

状态：等待用户从三套方向中选择
更新时间：2026-07-16

## 共同约束

- Android 390 × 844 基准画布，遵循 Material 3 交互尺寸和系统返回行为。
- 月历是首屏主角，日期详情采用底部面板。
- 咖啡与健身必须能在同一天同时辨识。
- 颜色不是唯一信息载体；同时提供图标、点形、文字或数量。
- 常用记录操作在 3 次点击内完成。
- 视觉方向只改变表达，不改变状态与统计规则。

## 方向 1：暖色纸感日志

- 浅米白背景、咖啡棕主色、训练绿辅助色。
- 更像私人纸质日历，强调舒适、长期和低压力。
- 日期单元格留白较多，活动以圆点和小数量徽标呈现。
- 适合“私人生活记录”定位；风险是过度温和导致统计对比不足。

## 方向 2：数据优先仪表盘

- 冷白或浅灰背景、蓝色主色，活动色更明确。
- 月历和统计密度更高，强调效率、精确值和趋势。
- 适合重视量化的用户；风险是首屏像工具表格，降低情感亲和力。

## 方向 3：彩色活动卡片

- 中性背景，手冲、健身和未来活动使用鲜明但可访问的主题色。
- 日历下方以大活动卡片提供 `+1` 和完成快捷操作。
- 适合未来扩展多活动；风险是活动增多后色彩和卡片密度上升。

## 选择标准

按以下顺序判断，而不是只选“最好看”：

1. 是否能在 5 秒内理解并完成一次手冲记录。
2. 是否能区分 `UNSET`、`MISSED`、`SKIPPED` 与 `DONE`。
3. 同一天有两项活动时是否仍清晰。
4. 周/月/年统计是否能保持可信和易读。
5. 新增第三、第四种活动时是否仍可扩展。

## 选定后的低保真产出

用户选择方向后，在同一 Figma 文件创建四个可点击画面：

1. 月历首页。
2. 日期详情底部面板。
3. 统计总览。
4. 活动管理。

原型完成后用 5 个任务走查：新增一次手冲、修改次数、标记健身没做、查看本月统计、归档活动。
