# 开发准备度基线

状态：Ready for local MVP implementation after M0 validation
更新时间：2026-07-16

本文冻结 Daily Record 从产品发现进入 Android 本地 MVP 前的实现边界。视觉原型、数据模型、统计口径和 GitHub Issue 发生冲突时，应先修正文档并记录决策，不在页面代码中临时发明规则。

## 1. P0 冻结范围

P0 包含：

- 不登录也能使用的本地活动、每日记录、月历和精确统计。
- `COUNT`、`BOOLEAN`、`DURATION` 三种基础计量类型。
- `UNSET`、`DONE`、`MISSED`、`SKIPPED` 四种记录状态。
- 活动创建、编辑、排序和归档；归档保留历史记录与统计。
- 周、月、年和全部历史周期；活动筛选与时间周期互相独立。
- 0/2/4/8+ 活动下稳定的日历汇总胶囊。
- COUNT 精确输入、DURATION 5 分钟步进与精确输入，以及本地保存的失败/重试/撤销反馈。

P0 不包含：强制登录、云同步、目标与连续天数、提醒、广告、付费墙、任意日期独立颜色、永久删除活动、端到端加密、深色主题和高级趋势图。

## 2. 数据与统计契约

- `MeasurementType = COUNT | BOOLEAN | DURATION`；手冲、健身等名称是 Activity 实例，不是枚举或专属表。
- `RecordStatus = UNSET | DONE | MISSED | SKIPPED`。
- `COUNT` 大于 0 时为 `DONE`，等于 0 时为 `MISSED`；清除记录才回到 `UNSET`。
- `UNSET` 与 `SKIPPED` 的 quantity 为空；`BOOLEAN` 以 status 为业务事实。
- 同一用户、活动和本地日期最多一条 DailyRecord，唯一约束为 `(ownerId, activityId, localDate)`。
- 单活动筛选约束整页统计；时间范围中的“全部”只表示全部历史，不表示全部活动。
- 表格与汇总卡必须来自同一查询结果。固定视觉目标为总次数 128、活动天数 74、记录日均 1.7；当前统计 PNG 的月次数合计为 118、月天数合计为 78，尚未达到契约。[#14](https://github.com/LitaoG/daily-record-app/issues/14)给出修订 fixture、演示参考日期与未来月份规则。

## 4. 边界行为

- 已保存的 `localDate` 不因时区改变而漂移；周起始日默认星期一，可切换为星期日。
- 活动重名允许但必须依靠图标和颜色辨识；颜色不是状态的唯一编码。
- 活动基础色跨页面保持一致；空日期默认不渲染 `0` 汇总胶囊。
- 有历史记录的活动不能直接改变 measurementType；必须归档旧活动并新建活动，或在后续版本提供显式迁移。
- 清除记录和归档活动属于可逆操作；P0 不提供轻易永久删除入口。
- 所有写入先落本地；网络、登录或未来同步失败不得回滚用户已看到的数据。

## 5. 数据安全与可携带性

- 日志、分析事件、公开截图和示例数据不得包含真实活动名、备注、数值、账号令牌或数据库文件。
- P1 导出至少提供版本化 JSON 备份和可读 CSV；导入必须先校验版本、所有权、重复项和计量类型，再在事务中落库。
- 云端区域、账户合并、冲突恢复、删除账户和端到端加密在本地 MVP 稳定后通过独立 ADR 决策。
- 正式发布前必须补齐隐私政策、应用商店数据安全声明、签名密钥管理和第三方许可证清单。

## 7. Figma 审核通过后的独立阶段

- 重新授权并验证原生写入权限，先复制现有文件作为安全备份。
- 先建立颜色、字体、间距、圆角和状态变量，再创建日期格、汇总胶囊、活动行、统计卡和导航组件。
- 以仓库五张 PNG 为结构基线、Markdown 与 ADR 为业务事实来源，使用 390 × 844 画布逐屏迁移并做并排视觉 QA。
- 如果 Figma Make 能被当前工具调用，只为月历和统计页生成三个优化方向；用户选择后才写回正式组件。
- Make 不能改变已冻结的数据、状态、筛选和统计规则，也不能覆盖原始评审基线。

## 8. 进入开发与发布门槛

- 本地分支与 Draft PR head 一致，工作区干净且有恢复分支。
- Markdown 链接、`git diff --check`、单元测试和 Android lint 通过。
- 固定统计数据集正确率 100%，不可恢复数据丢失为 0。
- 统计原型的月次数、月天数、汇总卡和趋势图使用同一固定数据集，合计与 128 / 74 / 1.7 一致。
- 五人测试达到：核心任务完成率至少 90%、常用记录中位用时不超过 5 秒、统计理解率至少 80%。
- 原型审计中的活动色/状态色冲突、空日期、输入和错误反馈阻断项已关闭。
- 上述门槛满足前，PR #3 保持 Draft；Figma 权限不阻塞本地数据模型准备。
