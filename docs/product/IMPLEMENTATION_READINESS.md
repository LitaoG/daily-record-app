# 实现准备度

## 已冻结

- 唯一对象：手冲。
- 每个日期最多一条记录。
- 次数为非负整数；0 表示明确没冲；无记录表示未填写。
- 主导航仅日历与统计。
- 统计周期：周、月、年、全部历史。
- 指标：总次数、手冲天数、记录日均。
- 本地 Room 是唯一 UI 事实来源；邮箱密码账号和云同步可选。
- 登录后本机数据合并到账号；新设备可恢复，普通登录无短信验证码。

## 工程门槛

- Room v3 schema 导出并纳入版本控制。
- v1→v2→v3 与 v2→v3 迁移测试通过，旧手冲数据不丢失。
- 单元测试、Lint、Debug/Release 编译、设备数据库/Compose 和 Firestore 规则测试通过。
- 生产 Firestore 采用 `asia-east1` 生产模式，账户隔离规则已发布。
- Figma 页面不出现活动筛选、健身或活动管理。
- 0 次、未填写、1 次、2 次和 9+ 状态完成视觉与 TalkBack 规范。

## 产品门槛

- 五人测试核心任务完成率 ≥90%。
- 常用记录中位用时 ≤5 秒。
- 统计理解率 ≥80%。
- 没有用户把 0 次误解为未填写。

## 当前状态

- 当前展示名称固定为“手冲日历”；技术包名保持 `io.github.litaog.dailyrecord`，避免无必要迁移。
- Figma v2、Compose 日历/记录/统计、历史日期跳转、Room v3、邮箱密码登录和云同步基础设施均已实现。
- 当前事实资产只保留在 `assets/hand-brew-v2/`；旧多活动 PNG 已从当前分支删除，仍可通过 Git 历史审计。
- GitHub 连接暂不可用；本地分支与提交作为唯一变更记录，恢复后再推送。
- 发布前仍需五人可用性测试、两台实体设备生产同步回归、App Check、正式隐私政策、数据删除流程与签名 AAB。
- 当前网络的 Firebase 生产烟雾测试需代理才通过；中国大陆无代理云恢复仍是明确发布门槛，不影响本地优先 MVP。
