- 修复迟到上传确认覆盖新编辑、离线旧设备覆盖较新记录、设备时钟回拨导致版本变旧和跨账号缓存串读风险。
- 生产项目 `daily-record-hand-brew` 已建立：Email/Password 已启用，Firestore Standard 生产库位于 `asia-east1`，账户隔离规则已发布。
- 年月标题和统计周期共用同一浏览日期；月周明细在换月后重新计算，明确 0 次与未填写保持不同。

- 本机模式选择改为跨冷启动持久化；用户点击“登录同步”时明确退出该模式。
- Firestore 实时监听在瞬时错误后指数退避并自动重新订阅，避免一次断线永久失去跨设备更新。
- 午夜刷新改用 Android 8 可用的时区 API，并覆盖夏令时边界，保持 `minSdk 26` 兼容。
- 日历、记录页和统计汇总统一使用 `Long` 累加，防止多天高次数把总数溢出为负值。
- Firestore 规则与客户端解析拒绝超出公历 9999 年的毫秒时间戳，避免损坏数据毒化后续本机编辑。
