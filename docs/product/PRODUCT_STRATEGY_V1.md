# Product Strategy v1

状态：Accepted for MVP planning
更新时间：2026-07-16

## 1. 一句话价值

打开日历，3 秒内记录今天做过什么；之后能可信地看到行为在周、月、年如何变化。

## 2. 目标用户与核心任务

### 目标用户

- 想用简单日历记录私人行为的 Android 个人用户。
- 同时需要次数型活动（如手冲次数）和完成型活动（如健身天数）的用户。
- 希望无网也能使用，未来登录其他设备后可以恢复数据的用户。

### Jobs to be Done

1. 当我刚完成一项活动时，我希望在 5 秒内记一笔，不打断正在做的事情。
2. 当我回看某天时，我希望一眼知道做过什么、没做什么、做了几次。
3. 当我查看统计时，我希望数字能追溯到每天的原始记录，且周、月、年口径一致。
4. 当我增加新的记录主题时，我希望只新增活动配置，不需要换一套应用。

## 3. 产品原则

- **快于全面**：常用记录路径不超过 3 次点击，复杂编辑放在二级入口。
- **本地优先**：网络不是记录动作的前置条件，Room 是单一事实来源。
- **状态明确**：没有记录不等于明确没做，跳过不等于失败。
- **统计可信**：所有统计由原始记录重算；UI 不自行创造口径。
- **扩展靠配置**：手冲和健身都使用通用 `Activity` 模型。
- **默认隐私**：核心功能无需登录；敏感内容不进入公开日志或默认遥测。

## 4. MVP 产品闭环

### P0：本地可用闭环

- 内置手冲（次数型）与健身（完成型）模板。
- 创建、编辑、排序、归档通用活动，并配置颜色、图标和计量方式。
- 月历作为主入口，点击日期打开日期详情。
- 次数型支持 `+1`、`-1`、直接输入和清除。
- 完成型支持完成、没做、跳过和清除。
- 展示总次数、总活动天数，以及周、月、年精确表。
- 支持活动筛选与全部历史周期，两个作用域互相独立。
- COUNT 数值可精确输入；DURATION 默认 5 分钟步进并可精确输入。
- 本地保存提供失败、重试和撤销反馈。
- 飞行模式下可新增、修改和回看记录；重启后数据仍存在。

### P1：可回顾与可携带

- 趋势图和年度热力图。
- CSV/JSON 导出与导入校验。
- 提醒和桌面小组件。
- 深色模式、大字号与 TalkBack 完整检查。

### P2：账户与同步

- 邮箱登录、游客数据绑定、多设备恢复。
- Outbox、后台重试、同步状态和冲突说明。
- 行级权限、账号删除和云端数据删除。

### 当前明确不做

- 社交、排行榜、好友共享。
- 复杂训练计划、动作组数和医疗建议。
- AI 教练、广告、订阅和付费墙。
- iOS、Web、Wear OS；启动前需单独评估。

## 5. 关键状态与颜色规则

| 状态 | 用户含义 | 次数型 quantity | 是否计入活动天数 |
|---|---|---:|---:|
| `UNSET` | 用户未表达 | `null` | 否 |
| `DONE` | 已完成 | 正整数或完成型隐含 1 | 是 |
| `MISSED` | 明确没做 | 0 | 否 |
| `SKIPPED` | 主动跳过 | `null` | 否 |

- 清除记录会回到 `UNSET`，不是 `MISSED`。
- 次数改为 0 时必须转为 `MISSED`，不能继续以完成颜色展示。
- 活动拥有基础色；全部活动概览中同日只显示一个汇总胶囊，最多 3 色段并显示总活动数。
- 只筛选单一活动时，可以用同一基础色的深浅表达次数强度。
- 颜色必须同时配合文本、图标、形状或可访问描述。

## 6. 阶段成功条件

P0 只有在以下条件同时满足时才算完成：

- 飞行模式下完成从记录到统计的完整路径。
- 固定测试数据集的总次数、总天数、周/月/年表全部正确。
- `UNSET`、`MISSED`、`SKIPPED` 在日历和日期详情中可区分。
- 常用记录路径中位完成时间不超过 5 秒。
- 归档活动保留历史统计且不再出现在默认快速记录区。
- 核心流程可以通过 TalkBack 完成，颜色不是唯一编码。
- 没有真实用户数据、密钥或敏感日志进入公开仓库。
