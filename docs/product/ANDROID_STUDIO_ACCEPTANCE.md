# Android Studio 人工验收清单

状态：工程自动化基线已通过，等待真实使用验收。

## 1. 测试准备

- 使用 Android Studio 打开仓库根目录，等待 Gradle Sync 完成。
- 确认 `app/google-services.json` 存在于本机但没有出现在 Git 变更中。
- 准备设备 A；云恢复验收另准备一台真机或一个全新的设备 B 模拟器。
- 不在截图、Issue 或提交中记录测试密码、Firebase 授权码或真实隐私数据。
- 设备日期、时区和字体缩放先保持日常设置；可访问性场景再单独调整。

## 2. 本地记录基线

1. 全新安装后选择“先在本地使用”。
2. 把今天保存为 2 次、昨天保存为 0 次、三天前保存为 1 次。
3. 确认 2 次显示为已手冲、0 次显示为明确没冲、未填写日期仍为空白。
4. 强制结束并重新打开应用，三种状态必须保持。
5. 修改三天前记录并连续快速点击保存，最终只能得到一次确定写入，界面不能卡在加载状态。
6. 清除一条记录并确认二次确认弹窗；取消时数据不变，确认后回到未填写。

## 3. 历史日期快速跳转

1. 在日历页点击大号年月标题，例如“2026 年 7 月”。
2. 日期选择器中直接选择另一个年份、月份和日期，例如 `2025-07-03`。
3. 返回后标题、选中日期和日期记录面板必须同时定位到 `2025-07-03`，不能仍停留在原月份。
4. 切到统计页，周统计应覆盖该日期所在自然周，月统计应为 2025 年 7 月，年统计应为 2025 年。
5. 在周/月/年之间切换后再回日历，浏览日期不能自动跳回今天。
6. 验证 1970 年 1 月下限、闰年 2 月、12 月到次年 1 月，以及未来日期不可记录。

## 4. 月份周明细重建

1. 在月统计中选择一个有记录的月份，记下周明细行数、日期范围和合计。
2. 切换到前一个或后一个月份。
3. 每一行必须重新显示新月份内的日期范围，不能保留上个月的周标题、次数或天数。
4. 周明细次数之和必须等于月汇总次数，周明细手冲天数之和必须等于月汇总天数。
5. 历史未填写显示“未填写”，明确 0 次显示 0；完全位于未来的日期保持空值而不是预测值。

## 5. 邮箱密码与云恢复

1. 设备 A 使用专用测试邮箱和密码注册，不使用短信或验证码。
2. 确认本机已有记录合并到账号空间，并最终显示同步完成。
3. 开启飞行模式新增或修改一条记录；本地界面必须立即更新并显示待同步。
4. 恢复网络，等待状态变为已同步，再强制结束并重启应用确认数据仍在。
5. 在全新的设备 B 安装应用并登录同一账号，确认新增、修改、明确 0 次和已清除记录全部恢复，统计与设备 A 一致。
6. 设备 B 退出后注册第二个测试账号，不能看到第一个账号的任何记录。
7. 再次登录第一个账号，数据应重新恢复且不重复累计。

## 6. 交互与可访问性

- 旋转屏幕后，正在浏览的日期、统计周期、已输入但未保存的次数和弹窗状态符合预期。
- 字体缩放到 200% 后，年月标题、保存按钮、统计卡和周/月/年表格仍可读可点，无文字重叠或关键内容截断。
- TalkBack 能读出日期、次数、未填写/没冲/已手冲状态，以及按钮用途；颜色不能成为唯一状态提示。
- 快速双击保存、保存中返回、保存失败、清除失败、断网重连都不能造成重复记录或永久禁用按钮。

## 7. 通过标准与记录格式

以下任一情况都算失败，不能用“重新打开就好了”作为通过理由：

- 历史跳转后日历与统计使用不同日期。
- 切月后周明细串月、合计不一致或未来月份出现预测值。
- 0 次与未填写混淆。
- 离线编辑丢失、重复累计、清除记录复活或不同账号互相看到数据。
- 登录/同步在目标实际网络中不可用且没有明确错误提示与本地使用退路。

发现问题时记录：设备/系统版本、应用提交号、操作前状态、精确步骤、预期、实际、截图或录屏、是否可稳定复现。当前验收提交基线为 `b4a4e68` 之后的最新提交。

## 8. 发布前仍需单独完成

人工功能验收通过不等于可以直接上架。正式发布前仍需配置发布签名与密钥保管、启用匹配正式签名的 App Check、补齐隐私政策和账号/云数据删除入口，并验证目标地区真实网络下 Firebase 的可达性。
