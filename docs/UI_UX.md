# 界面与交互

## 导航结构

底部主导航建议包含：

1. 日历
2. 统计
3. 活动
4. 设置

快速记录优先从日历日期详情或全局悬浮按钮进入，原型测试后确定最终入口。

## 日历页

- 顶部：月份标题、前后切换、回到今天。
- 中部：星期标题和月份网格。
- 日期状态分层：今天轮廓、选中背景、活动色点互不覆盖。
- 单日最多直接显示三个活动颜色，更多显示 `+N`。
- 可按活动筛选，筛选后次数型活动可用颜色强度表示次数。
- 空白日期和明确未完成必须视觉不同。

## 日期详情

使用底部抽屉或独立页面展示：

- 日期标题和备注入口。
- 每个活动的颜色、名称、状态和数量。
- 次数型：`-`、数值、`+`，以及直接输入。
- 完成型：完成、未完成、跳过、清除。
- 保存使用本地即时提交；不显示阻塞式网络加载。

## 统计页

- 顶部选择活动和周期。
- 第一屏优先展示主指标，不堆叠过多卡片。
- 周/月/年表格提供精确值，图表用于观察趋势。
- 热力图必须配图例并支持色觉友好主题。
- 当前周期标记“进行中”。

## 活动管理

- 活动列表支持排序和归档。
- 新建/编辑包含名称、颜色、图标、计量方式、单位。
- 已有历史数据时改变计量方式需要限制或数据迁移提示。

## 隐私体验

- 应用锁在后台一段时间或冷启动时触发。
- 隐私模式可以隐藏敏感活动名称和通知内容。
- 最近任务缩略图是否模糊作为后续隐私选项评估。
- 导出和删除前清楚说明范围，不使用诱导性确认。

## 无障碍

- 颜色不是唯一信息载体，同时提供图标、形状或文本。
- 交互目标至少满足 Material 建议尺寸。
- 支持系统字体缩放和 TalkBack 描述。
- 深色模式检查颜色对比度。
- 图表提供摘要或可访问数据表。

## 原型顺序

1. 日历空状态与多活动颜色。
2. 日期详情和快速 `+1`。
3. 次数型与完成型混合记录。
4. 周/月/年统计。
5. 登录、同步状态和隐私设置。
