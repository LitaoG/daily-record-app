# 本地数据与隐私

当前产品不包含账号、登录、云同步或第三方分析。所有手冲记录只保存在设备内的 Room 数据库中，核心功能不请求网络权限。

## 数据最小化

每条记录只保存日期、手冲次数、创建时间和更新时间。不保存位置、设备标识、联系人、健康数据或活动备注。

## 备份说明

当前 Manifest 设置 `android:allowBackup="false"`，并在 API 26–30 的 `backup_rules.xml` 与 API 31+ 的 `data_extraction_rules.xml` 中显式排除数据库、文件、偏好和外部文件。系统云备份和设备迁移均不会复制手冲记录。

卸载应用或清除应用数据会永久删除本地记录。当前版本不承诺跨设备恢复，也不把“未来同步”作为数据模型设计目标；如果以后改变此策略，必须先新增清晰的用户说明、导出/恢复方案和隐私验收。

## 公开仓库

禁止提交真实数据库、屏幕录制中的私人记录、密钥、签名或账号信息。测试只使用固定虚构数据。
