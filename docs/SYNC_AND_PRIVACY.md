# 同步与隐私

应用坚持本地优先：未登录也能完整记录、查看日历和统计；登录只用于跨设备同步与恢复。所有界面数据来自设备内 Room，网络失败不会阻塞保存。

## 账号

- 仅提供邮箱和密码注册/登录，普通登录不使用短信或验证码。
- Firebase UID 是账号所有权边界；本机 Room 也按 UID 隔离缓存。
- 密码由 Firebase Authentication 处理，应用和 Room 不保存明文密码。
- 忘记密码使用 Firebase Authentication 向输入邮箱发送重置链接；应用不会读取、生成或保存新密码。
- 重置请求对已注册和未注册邮箱显示相同成功提示，避免通过界面枚举账号；断网、限频与额度错误只显示可执行的净化提示。
- 登录时，`__local__` 本机记录会合并到这次登录的账号；界面在操作前明确提示这一点。
- 用户明确选择本机模式后会跨冷启动保留该选择；点击顶部“登录同步”会清除该偏好并返回登录页。

## 云端数据

Firestore 路径为 `/users/{uid}/handBrewRecords/{YYYY-MM-DD}`。每个文档只含稳定 ID、本地日期、次数、创建/修改时间、删除墓碑、修订号、schema 版本和服务器更新时间。不保存位置、联系人、健康数据、备注或设备标识。

生产数据库位于 `asia-east1`，以生产模式创建。安全规则要求：

- 必须已登录且 UID 与路径一致。
- 字段必须完全匹配白名单，次数非负且日期与文档 ID 一致。
- ID、本地日期和创建时间在更新后不可改变。
- 修订号每次只增加 1；客户端不能物理删除文档。

## 离线、冲突与删除

- 本地修改先写 Room 并标记 `PENDING`；联网、应用启动、手动同步或 WorkManager 重试时上传。
- 远端快照不能覆盖尚未确认的本地版本；迟到的上传确认也不能把新编辑标成已同步。
- 两台设备编辑同一天时，较新的 `clientUpdatedAt` 获胜；完全相同时间保留服务器版本。
- 清除记录写入墓碑，防止另一台长期离线设备把旧记录恢复出来。
- Firestore 实时监听遇到瞬时错误会指数退避并自动重新订阅；离线期间等待有效网络，不进行无意义重试。
- 登录后的本机记录迁入只依赖 Room，不等待 Firestore；服务器监听恢复且仍有 `PENDING` 数据时会立即触发补偿同步，即使系统网络状态没有变化。
- 同步失败只显示经过净化的错误，不在日志或 UI 暴露文档内容。

## 设备备份

Manifest 保持 `android:allowBackup="false"`，系统云备份和系统设备迁移不会复制数据库。跨设备恢复只通过用户主动登录同一账号完成。未登录且从未同步的本机记录在卸载或清除应用数据后无法恢复。

## 发布前隐私门槛

- 提供可访问的隐私政策与账号/云数据删除入口。
- 用正式签名配置并执行 App Check 评估后再启用强制执行。
- 复核 Firebase 数据保留、支持邮箱、滥用配额与告警。
- 禁止提交真实数据库、账号口令、服务账号、签名文件或 `google-services.json`。
