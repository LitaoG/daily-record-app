# 产品契约：手冲日历

状态：Hand-brew-only v1 冻结
最后更新：2026-07-17

## 产品目的

让用户在几秒内记录“某天手冲了几次”，并能可信地回看日历以及周、月、年和全部历史统计。

产品只有一个记录对象：手冲。界面中没有活动选择器、活动管理、健身指标或新增其他记录类型的入口。

## P0 用户任务

1. 打开应用查看当前月历。
2. 点击日期，使用 `+1`、`-1` 调整并保存该日手冲次数。
3. 把次数设为 `0`，明确表示当天没冲。
4. 清除记录，让日期回到“尚未填写”。
5. 查看周、月、年和全部历史的次数、天数、日均与精确明细。
6. 点击年月标题直接选择历史年份和日期，不需要逐月点击箭头。

## 业务规则

- 每个本地日期最多一条记录。
- 次数只接受非负整数。
- `count > 0`：已手冲。
- `count = 0`：明确没冲。
- 没有数据库行：尚未填写。
- 总次数是范围内 `count` 之和。
- 手冲天数只统计 `count > 0` 的不同日期。
- 记录日均 = 总次数 / 手冲天数；没有手冲天数时显示 `0.0`。
- 未来日期默认不可记录；历史日期可编辑。
- 日历支持 1970 年 1 月 1 日至今天，最早月份继续向前的导航禁用。
- 日历和统计共享同一个“浏览日期”；从日历切到统计不会跳回今天，周/月/年切换也不会重置浏览日期。
- 所有保存先落本地数据库，界面立即更新。

## 主界面

- 日历：默认入口，查看每一天的手冲状态和次数。
- 统计：查看周、月、年和全部历史汇总及明细。
- 日期记录面板：从日历进入，不作为第三个主导航。

## 明确不做

健身、喝水、学习、睡眠、跑步、自定义活动、活动颜色配置、活动管理、目标、连续打卡、提醒、社交、排行榜、登录、云同步、跨设备恢复、订阅、广告和 iOS/Web 客户端均不属于当前产品。

## P0 验收

- 飞行模式下可新增、修改、清除记录，重启后仍存在。
- 同一天重复保存只更新同一条记录。
- `0` 与未填写在数据库、视觉和 TalkBack 语义上不同。
- 固定数据集正确得到 128 次、74 天、日均约 1.7。
- 周/月/年/全部历史的汇总与明细加总一致。
- 跳转历史日期后，周统计取该日期所在的自然周，月统计取该日期所在月，年统计取该日期所在年。
- 闰年、月末、跨年周和空数据不崩溃、不伪造数据。
