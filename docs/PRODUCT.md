# 产品需求文档

状态：Product Discovery v1 baseline
最后更新：2026-07-16

> 本文保留整体需求基线。目标用户、用户故事、KPI、研究证据与设计推进状态见 [产品发现索引](product/README.md)。

## 1. 产品愿景

Daily Record App 是一个以日历为中心的私人活动记录工具。用户可以用颜色区分活动，快速记录某天是否完成以及完成次数，并从周、月、年维度理解自己的行为。

第一批内置示例活动为：

- 手冲：主要使用“次数”计量，同时统计发生天数。
- 健身：第一阶段主要使用“是否完成/天数”计量。

产品必须允许未来增加活动和计量方式，而不需要复制数据库、统计页和同步逻辑。

## 2. 目标用户

- 希望用简单日历记录私人行为的个人用户。
- 需要同时记录次数型和完成型活动的用户。
- 希望离线记录并在多台 Android 设备之间同步的用户。

## 3. 核心原则

1. 记录动作应在几秒内完成。
2. 不登录也能使用核心功能；登录用于备份与同步。
3. 用户看到的统计必须能够从原始记录重新计算。
4. 多活动同日发生时，日历仍然清晰。
5. 隐私功能不能等到发布前才补救。
6. 扩展新活动优先使用配置，不增加成套重复页面和表。

## 4. MVP 范围

### 4.1 日历

- 月份前后切换和快速回到今天。
- 支持星期一或星期日作为一周起点，默认星期一。
- 今天和当前选择日期具有独立视觉状态。
- 日期单元格始终只显示一个活动汇总胶囊。
- 汇总胶囊最多显示三个活动色段，并显示当天活动总数；8 项及以上显示 `8+`。
- 点击日期打开当天记录面板。
- 支持查看没有记录、明确未完成、完成和跳过等状态。

### 4.2 记录

- 次数型活动支持 `+1`、`-1` 和直接输入非负整数。
- 完成型活动支持完成、未完成、跳过和清除记录。
- 支持可选备注；备注默认不出现在日历主界面。
- 修改历史日期与修改今天使用同一套规则。
- 记录保存后日历和统计立即更新，不依赖网络。

### 4.3 活动管理

- 创建、编辑、排序、归档活动。
- 配置名称、图标、主题色、计量方式和显示单位。
- 活动归档后保留历史记录和统计。
- 删除活动是高风险操作，MVP 默认只提供归档；永久删除以后实现。

### 4.4 统计

- 全部历史总次数和总活动天数。
- 当前周、月、年次数与活动天数。
- 任意周、月、年切换。
- 周表、月表、年度月份表。
- 年度日历热力图。
- 空数据状态和不足完整周期的解释。

### 4.5 账户和同步

- 游客模式先创建本地数据。
- 邮箱登录作为第一个跨设备身份方案；第三方登录另行决定。
- 登录后可选择把游客数据合并到账号。
- 展示同步状态、最近同步时间和错误重试入口。
- 退出登录前说明本机数据处理方式。

### 4.6 设置与隐私

- 浅色、深色、跟随系统。
- 一周起始日和语言/日期格式。
- 应用锁和生物识别重新授权。
- CSV/JSON 导出。
- 删除账号与云端数据的入口在同步版本中实现。

## 5. 非 MVP 范围

- iOS、Web 和桌面客户端。
- 社交、排行榜和好友共享。
- 医疗诊断或健康建议。
- 复杂训练动作、组数、重量和身体指标。
- 精确提醒、桌面小组件和 Wear OS。
- 订阅、付费和广告。
- 端到端加密；架构需保留升级空间，但是否实现需要单独决策。

## 6. 主要用户流程

### 首次使用

1. 打开应用并阅读简短隐私说明。
2. 选择游客模式或登录。
3. 从“手冲”和“健身”模板中选择，也可以创建自定义活动。
4. 进入当前月份日历。

### 快速记录一次活动

1. 点击今天或日历中的目标日期。
2. 在活动卡片中点击 `+1` 或完成。
3. 本地立即保存，统计同步更新。
4. 如果已登录，将变更放入后台同步队列。

### 查看统计

1. 进入统计页并选择活动。
2. 选择周、月、年或全部。
3. 查看汇总卡片、表格和趋势图。
4. 点击图表或日期返回对应日期记录。

## 7. 验收标准

- 飞行模式下可以创建和修改记录，重启应用后数据仍存在。
- 同一活动的总次数等于每日次数之和。
- 活动天数只计算次数大于零或状态为完成的不同本地日期。
- 切换月份后颜色和统计与记录一致。
- 闰年、跨年周和时区变化不会造成崩溃或静默丢失。
- 归档活动不再出现在默认快速记录区，但历史统计仍可查询。
- 公开仓库和应用日志中没有真实用户记录或秘密密钥。

## 8. 编码前待确认

- 正式应用名称与 Android 包名。
- 主要使用地区和云端部署区域。
- 是否预期一年内开发 iOS。
- “明确未完成”是否参与某些完成率统计。
- 是否需要独立的任意日期颜色标记，还是颜色只来源于活动。
- 是否第一版就需要端到端加密或仅使用平台与云服务静态加密。

## 9. Product Discovery v1 已确认

- P0 先交付离线记录闭环，账户与同步后移到 P2。
- 日历为默认主入口，主导航为日历、统计、活动、设置。
- `UNSET`、`DONE`、`MISSED`、`SKIPPED` 明确区分。
- 次数变为 0 时转为 `MISSED`；清除记录才回到 `UNSET`。
- 未定义目标分母前不展示完成率。
- 活动颜色只表达活动身份；多活动同日使用单个汇总胶囊、最多 3 色段和总活动数，颜色不是唯一编码。
