# 统计口径

统计必须使用清晰定义和固定测试数据。图表只展示查询结果，不在 UI 中重新计算指标。

## 基础定义

### 总次数

针对次数型活动：

```text
totalCount = SUM(quantity)，排除 deletedAt 非空的记录
```

### 活动天数

不同本地日期的数量，满足以下任一条件：

- 次数型活动：`quantity > 0`。
- 完成型活动：`status == DONE`。

`MISSED`、`SKIPPED` 和 `UNSET` 不计入活动天数。

### 周、月、年

- 周：依据用户的周起始设置生成连续 7 天。
- 月：自然月。
- 年：自然年 1 月 1 日至下一年 1 月 1 日之前。
- 所有查询使用 `[startDate, endExclusive)`。

## MVP 指标

| 指标 | 次数型 | 完成型 |
|---|---:|---:|
| 总次数 | 是 | 可显示完成次数，但不作为主指标 |
| 总活动天数 | 是 | 是 |
| 周/月/年次数 | 是 | 可选 |
| 周/月/年活动天数 | 是 | 是 |
| 日均次数 | 有活动天数内总次数 / 活动天数 | 不适用 |
| 周期完成率 | 需要目标定义后再做 | 需要目标定义后再做 |

没有配置目标之前，不展示“完成率”，否则分母不明确。

## 表格

- 周表：7 行或 7 列，展示日期、状态、次数。
- 月表：按日期展示，附本月合计。
- 年表：12 个月，每月展示次数和活动天数。
- 热力图：颜色强度按当前活动的日 quantity 分桶；完成型使用二值强度。

## 边界与展示

- 当前未结束周期必须标记为“进行中”。
- 无数据展示 `0` 和解释性空状态，不用 `--` 混淆“未知”。
- 归档活动仍参与其历史范围统计。
- 软删除记录不参与任何统计。
- 时区改变后，已保存的 `localDate` 不自动移动。
- 大数显示可以格式化，但底层计算保持完整精度。

## 测试矩阵

- 0 条记录。
- 同一天多次修改 quantity。
- 2 月 29 日。
- 月末到下月。
- 12 月末到次年。
- 星期一和星期日为周起始。
- 跨年周。
- 归档活动。
- 软删除后重算。
- 时区从 UTC+8 切换到 UTC-8。

## 后续指标

连续天数、最长连续天数、目标完成率和趋势比较必须先定义：跳过日是否打断连续、明确未完成如何处理、周期目标如何设定。定义确定后再进入此文档和代码。
